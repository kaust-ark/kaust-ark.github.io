为什么选择 ARK

科研不该全自动

很多系统把科研当成固定任务，但真实科研并非如此。

01

目标天生模糊

研究想法很少一步到位。做到一半的发现能推翻一切 — 问题、方法、范围全变。

02

路径决定论文

两条路通向同一结果，写出来的论文截然不同。近似算法和 GPU 加速都能"解"矩阵问题 — 但那是两种科学。

03

人在成长

一次讨论、一个新发现，甚至一次复盘，都可能改变研究方向。你的工具也要能及时跟上。

我们的思路

是助手，不是替身

你负责决策，Agent 负责执行。

✨

分工协作

实验、LaTeX 迭代、文献调研、图表生成等重复工作，可交给专业 Agent 处理。

👁

可观测

每个 Agent 的动作、评分历史和关键决策都可追踪，不是黑箱。

☸

可干预

运行过程中可随时通过 Telegram 或网页注入新指令；关键分叉点 ARK 会暂停并等待你的决策。

掌舵机制

1

进度实时可见

每轮迭代实时记录 — Agent 动作、审稿评分、决策，网页门户随时看，Telegram 随时推。跑到哪了，一目了然。

2

关键节点主动请示

分数停滞、实验分支变化较大、计划存在歧义时，ARK 会先通知你并等待确认，再继续执行。

3

随时干预

通过 Telegram 或网页随时发送指令，ARK 会在下一轮迭代自动吸收你的新方向，无需重启流程。

工作原理

三阶段，自动迭代到达标

研究 → 开发 → 审稿，持续迭代直至达到目标分数。

1

研究

Deep Research 搜文献、理背景。

2

开发

定方案、跑实验、分析结果、写初稿。

3

审稿

迭代优化，直到审稿分数达到阈值。

审稿循环

每轮迭代 5 步

编译 LaTeX → PDF，页面图像
→
审稿 评分并识别问题
→
规划 优先处理行动项
→
执行 研究、实验、写作
→
可视化 修复图表并重新编译

认识团队

8 个 AI Agent，各司其职

每个 Agent 精通一个领域，通过共享状态和结构化交接协作。

Reviewer

给论文打分，列出具体改进任务。

Planner

把审稿意见转化成优先级行动计划。

Writer

按计划起草和打磨 LaTeX 各章节。

Experimenter

在远程算力上跑实验、收结果。

Researcher

查文献、找空白、搜证据。

Visualizer

生成符合会议排版规范的出版级图表。

Meta-debugger

识别流程停滞、诊断故障，并触发自修复。

Coder

编写代码、修复 bug、维护实验脚本。

快速开始

3 条命令即可开始

克隆、安装、建项目 — 剩下的交给 ARK。

# 克隆并安装
git clone https://github.com/kaust-ark/ARK.git && cd ARK
pip install -e .

# 创建项目（交互式向导）
ark new mma

# 运行 — ARK 开始执行
ark run mma

# 查看运行情况
ark monitor mma
ark new mma --from-pdf proposal.pdf ark status mma ark setup-bot pip install -e ".[research]"
