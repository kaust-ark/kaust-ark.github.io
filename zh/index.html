其他工具
  • 自动化漂移
    无人监督运行，中途无法纠偏。
  • 排版混乱
    布局错乱、LaTeX 报错，需大量人工修复。
  • 引用幻觉
    LLM 编造看似合理但不存在的参考文献。
  • 图片质量低
    默认样式、尺寸失控，无视栏宽和页面约束。
  • 配置繁琐
    需手动配置环境、调整依赖，上手门槛高。
ARK
  • 人机协同
    关键决策点暂停，随时通过 Telegram 或网页介入。
  • 出版级排版
    硬编码 LaTeX + 会议模板（NeurIPS、ACL、IEEE……）。
  • 引用验证
    每条引用经 DBLP API 验证，杜绝虚假文献。
  • 精准控图
    Paper Banana + 会议画布尺寸、栏宽、字号精确匹配。
  • 一键启动
    网页端一键运行，零配置开箱即用。

工作原理

ARK Pipeline

研究、开发、审稿——持续迭代直至达到目标分数。

1

研究

Deep Research 搜文献、理背景。

2

开发

定方案、跑实验、分析结果、写初稿。

3

审稿

迭代优化，直到审稿分数达到阈值。

认识团队

ARK Agents

八个专家 Agent，通过共享状态和结构化交接协作。

Reviewer

按会议标准给论文打分，生成改进任务。

Planner

将审稿意见转化为含实验和写作任务的行动计划。

Writer

撰写和打磨 LaTeX 章节，引用经 DBLP 验证。

Experimenter

设计实验、提交 Slurm 任务、分析结果。

Researcher

深度文献调研，通过学术 API 发现空白、搜集证据。

Visualizer

基于 Paper Banana 和会议画布尺寸生成出版级图表。

Meta-Debugger

识别流程停滞、诊断故障，触发自修复。

Coder

编写和调试实验代码与分析脚本。
