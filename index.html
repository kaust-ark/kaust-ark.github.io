Why ARK

Research should not be fully automated

Existing systems treat research like a task with a fixed problem. It isn’t.

01

Goals are ambiguous

A research idea rarely comes fully formed. Discoveries mid-way change everything — the question, the method, the scope.

02

Path matters

Two paths to the same result make very different papers. An approximate algorithm and a GPU-accelerated one both “solve” matrix penalty — but they’re not the same science.

03

Researchers evolve

A conversation with your advisor, a new paper, a fresh morning — research direction changes. Your system needs to change with you.

Our Approach

A harness, not a replacement

You hold the reins. Agents do the running.

✨

Offload

Delegate experiments, LaTeX iterations, literature review, and figure generation to specialized agents.

👁

Observe

Real-time visibility into every agent action, score history, and decision. Never a black box.

☸

Steer

Inject new directions mid-run via Telegram or web. At critical forks, ARK pauses and asks. Anytime override, always.

How Steering Works

1

Progress Awareness

Every iteration is logged in real time — agent actions, reviewer scores, and decisions are visible in the web portal and streamed via Telegram. You always know where the run is.

2

Key-Decision Notification

At critical forks — a stalled score, a major experiment branch, an ambiguous plan — ARK pauses and sends you a notification before proceeding. No silent decisions.

3

Anytime Intervention

Send a new instruction at any point via Telegram or the web portal. ARK incorporates your direction into the next iteration without restarting the run.

How It Works

Three-Phase Pipeline

ARK iterates through research, development, and review until the paper meets quality targets.

1

Research Phase

Deep Research gathers background knowledge and literature survey.

2

Development Phase

Plan experiments, run on compute, analyze results, evaluate completeness, write initial draft.

3

Review Phase

Iterative paper improvement loop until the reviewer score hits the acceptance threshold.

Review Phase

Each iteration runs 5 steps

Compile LaTeX → PDF, page images
→
Review Score & identify issues
→
Plan Prioritize action items
→
Execute Research, experiment, write
→
Visualize Fix figures & recompile

Get Started

Up and Running in 3 Commands

Clone, install, create a project, and hand the labor off to ARK.

# Clone & install
git clone https://github.com/kaust-ark/ARK.git && cd ARK
pip install -e .

# Create a project (interactive wizard)
ark new mma

# Run — ARK takes it from here
ark run mma

# Watch it work
ark monitor mma
ark new mma --from-pdf proposal.pdf ark status mma ark setup-bot pip install -e ".[research]"
